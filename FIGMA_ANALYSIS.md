## 📱 전체 프레임 구조

총 9개의 주요 프레임 (모바일 360x800)

```
Phase 1: 웹소설 읽기
├─ Frame 1: 읽기 뷰어 (기본)
├─ Frame 2: 로그 보기 활성화 (마커 표시)
└─ Frame 3: 마커 클릭 시 상세 뷰

Phase 2: 텍스트 스크랩 및 AI 생성
├─ Frame 4: 텍스트 선택 모드
├─ Frame 5: AI 생성 타입 선택 툴바
├─ Frame 6: 이미지 생성 바텀시트 (초기)
├─ Frame 7: 이미지 생성 중 (로딩)
└─ Frame 8: 이미지 생성 완료

Phase 3: 로그 갤러리
└─ Frame 9: 로그 모음 뷰
```

## 🔍 프레임별 상세 분석

### Frame 1 (68:158) - 웹소설 읽기 뷰어 (기본)

**기능**: 기본 웹소설 읽기 화면

#### UI 구성
- **상단 바** (Appbar/Context)
  - 좌측: 뒤로가기 아이콘
  - 중앙: "백작가의 맏나나가 되었다" (소설 제목)
  - 우측: "로그 보기" 체크박스 (비활성화)

- **본문 영역**
  - 웹소설 텍스트 표시
  - 스크롤 가능
  - 여백: 좌우 16px

- **하단 진행률 바**
  - 진행 바 (그래프)
  - 텍스트: "74 / 134" (현재 페이지 / 전체 페이지)
  - 우측: "48%" (진행률 퍼센트)

- **하단 네비게이션**
  - 3개 탭: 댓글 / 로그 / 설정
  - 각 탭에 아이콘 + 텍스트

#### 기능 매핑
- **4.1 웹소설 플랫폼**: 읽기 뷰어, 진행률 트래킹

---

### Frame 2 (68:2316) - "로그 보기" 활성화 상태

**기능**: 다른 독자들의 시각화 위치를 마커로 표시

#### UI 구성 (Frame 1과의 차이점)
- **우측 상단**: "로그 보기" 체크박스 **활성화** ✅
- **본문 위**: **원형 마커** 3개 표시
  - 위치: 본문의 특정 단어/문단 근처
  - 마커 = 다른 독자가 해당 위치를 시각화했다는 표시
  - 마커가 많을수록 인기 있는 장면

#### 기능 설명
- "로그 보기" 토글 시 본문에 **소셜 레이어** 표시
- 다른 독자들의 활동을 히트맵처럼 시각화
- 마커 클릭 시 → Frame 3으로 이동

#### 기능 매핑
- **4.4 로그 (Log)**: 로그 보기 활성화, 마커 표시

---

### Frame 3 (68:2610) - "{name}의 로그" 상세 뷰

**기능**: 마커 클릭 시 해당 독자가 생성한 시각화 콘텐츠 표시

#### UI 구성
- **배경**: 본문이 흐릿하게 표시 (dimmed)
- **바텀시트**
  - **상단 Grabber**: 드래그 핸들
  - **제목**: "{name}의 로그" (해당 유저 닉네임)
  - **우측**: "닫기" 버튼
  - **이미지 영역**: AI 생성 캐릭터 이미지 (붉은 머리 한복 캐릭터)
  - **우측 하단**: 하트 아이콘 (좋아요/저장)
  - **하단**: 스크랩된 원본 텍스트 미리보기

#### 사용자 액션
- 이미지 확대/축소
- 하트 클릭 (좋아요)
- 바텀시트 드래그로 닫기
- "닫기" 버튼 클릭

#### 기능 매핑
- **4.4 로그 (Log)**: 다른 독자 시각화 감상, 반응 (좋아요)

### Frame 4 (68:1386) - 텍스트 선택 모드

**기능**: 스크랩을 위한 텍스트 선택 준비 상태

#### UI 구성
- **본문만 표시** (UI 간소화)
- **우측 하단**: 커서 아이콘 (cursor-02)
  - 텍스트 선택 가능 상태를 시각적으로 표시
- 상단 바, 하단 네비게이션 숨김 (집중 모드)

#### 사용자 액션
- 본문 텍스트 드래그/선택
- 선택 완료 시 → Frame 5 플로팅 툴바 표시

#### 기능 매핑
- **4.2 텍스트 스크랩**: 텍스트 선택

---

### Frame 5 (68:1558) - AI 생성 타입 선택 툴바

**기능**: 텍스트 선택 후 나타나는 플로팅 액션 툴바

#### UI 구성
- **플로팅 툴바** (본문 위에 오버레이)
- **4개 옵션** (좌측부터):
  1. 📷 **이미지** (image-01 아이콘)
  2. 🎬 **비디오** (video-02 아이콘)
  3. 🎵 **음성** (audio-wave-01 아이콘)
  4. ✏️ **메모** (pencil-edit-01 아이콘)
- **구분선** (세로 라인)
- **X 버튼** (cancel-01) - 선택 취소

#### 사용자 액션
- 4개 옵션 중 하나 선택 → 해당 타입의 바텀시트 열림
- X 버튼 클릭 → 선택 취소, 툴바 사라짐

#### 기능 매핑
- **4.3 AI 멀티미디어 생성**: 생성 타입 선택

### Frame 6 (68:1762) - "이미지 만들기" 바텀시트 (초기)

**기능**: 이미지 생성 전 확인 화면

#### UI 구성
- **배경**: 본문 흐릿하게 표시
- **바텀시트**
  - **상단 Grabber**: 드래그 핸들
  - **제목**: "이미지 만들기"
  - **우측**: "닫기" 버튼
  - **안내 문구**: "스크랩한 글을 바탕으로 이미지가 생성됩니다."
  - **텍스트 박스**: 스크랩된 원본 텍스트 표시 (회색 박스)
  - **하단 버튼**: ✨ "만들기" (AI 마법 아이콘 ai-magic)

#### 사용자 액션
- 텍스트 확인
- "만들기" 버튼 클릭 → Frame 7 (로딩)

#### 기능 매핑
- **4.3 AI 멀티미디어 생성**: 생성 옵션 확인

### Frame 7 (68:1881) - "이미지 만들기" 바텀시트 (로딩)

**기능**: AI 이미지 생성 중 로딩 상태

#### UI 구성
- Frame 6과 동일한 레이아웃
- **이미지 영역**:
  - 로딩 스피너 (loading-03 아이콘)
  - 텍스트: "이미지를 생성중입니다..."
- **하단 버튼**: "만들기" (비활성화 또는 유지)

#### 사용자 경험
- 로딩 중 대기
- 예상 시간: 5~30초 (AI 모델에 따라)

#### 기능 매핑
- **4.3 AI 멀티미디어 생성**: 콘텐츠 생성 (프로세싱)

### Frame 8 (68:1954) - "이미지 만들기" 바텀시트 (완료)

**기능**: AI 생성 완료 후 결과 확인 및 액션 선택

#### UI 구성
- **이미지 영역**: **생성된 이미지 표시** (붉은 머리 한복 캐릭터)
- **하단 2개 버튼**:
  1. 📋 **"로그에 넣기"** (license-draft 아이콘)
     - 생성된 이미지를 로그에 저장
     - 다른 독자들과 공유
  2. 🔄 **"다시 만들기"** (reload 아이콘)
     - 같은 텍스트로 새로운 이미지 재생성
     - 다른 스타일/결과물 시도

#### 사용자 액션
- **로그에 넣기** 클릭:
  - 로그에 자동 저장
  - 바텀시트 닫힘
  - 해당 위치에 내 마커 추가
- **다시 만들기** 클릭:
  - Frame 7 (로딩)으로 돌아감
  - 새로운 이미지 생성

#### 기능 매핑
- **4.3 AI 멀티미디어 생성**: 결과 확인 및 저장
- **4.4 로그 (Log)**: 로그에 push

### Frame 9 (68:2041) - "로그" 갤러리 뷰

**기능**: 해당 소설의 모든 시각화 콘텐츠 모음

#### UI 구성
- **상단**:
  - Grabber (드래그 핸들)
  - 제목: "로그"
  - 우측: "닫기" 버튼
  - "내 로그만 보기" 체크박스

- **탭 메뉴** (4개):
  - **이미지** (선택됨)
  - 비디오
  - 음성
  - 메모

- **콘텐츠 그리드**:
  - 생성된 이미지들 (카드 형식)
  - 각 카드 우측 상단: "by 작업자" (작성자 표시)
  - 각 카드 하단: 스크랩된 원본 텍스트 미리보기

#### 사용자 액션
- 탭 전환 (이미지/비디오/음성/메모)
- "내 로그만 보기" 토글 (내 콘텐츠만 필터)
- 카드 클릭 → 상세 뷰 (Frame 3과 유사)
- 좋아요, 저장, 공유

#### 기능 매핑
- **4.4 로그 (Log)**: 로그 갤러리, 필터링, 탐색
- **소셜 기능**: 발견(Discovery), 커뮤니티 형성

## 📊 기술 구현 고려사항

### 프론트엔드
- **바텀시트 라이브러리** 필요 (드래그 가능)
- **텍스트 선택 UI** 커스터마이징
- **플로팅 툴바** 포지셔닝
- **마커 오버레이** 시스템
- **진행률 바** 실시간 업데이트

### 백엔드
- **로그 데이터 구조**:
  ```json
  {
    "novel_id": "소설 ID",
    "user_id": "유저 ID",
    "text_position": "텍스트 위치 (시작/끝)",
    "scrapped_text": "스크랩된 원본 텍스트",
    "content_type": "image|video|audio|memo",
    "content_url": "생성된 콘텐츠 URL",
    "created_at": "타임스탬프"
  }
  ```
- **마커 위치 계산**: 텍스트 오프셋 기반

### AI 통합
- **비동기 처리**: 생성 작업 큐
- **캐싱**: 같은 텍스트 재사용
- **스타일 옵션**: 추후 확장
