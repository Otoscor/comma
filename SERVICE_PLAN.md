# comma - 서비스 기획 문서

## 1. 서비스 개요

**comma**는 웹소설 독자를 위한 차세대 독서 플랫폼입니다. 텍스트로만 상상하던 소설 속 장면을 AI 기술을 통해 시각적, 청각적 콘텐츠로 구현하여 몰입감 있는 독서 경험을 제공합니다.

### 서비스명 의미
- **comma (,)**: 문장의 쉼표처럼, 독서의 흐름 속에서 잠시 멈춰 상상을 현실로 만드는 순간

---

## 2. 핵심 가치 제안 (Value Proposition)

### 문제 인식
- 웹소설 독자들은 텍스트만으로 장면을 상상해야 하는 한계
- 작가의 묘사와 독자의 상상 사이의 간극
- 몰입도를 높일 수 있는 멀티미디어 콘텐츠 부재
- 같은 소설을 읽는 독자들과 해석을 공유할 방법 부족

### 솔루션
**텍스트 스크랩 기반 AI 멀티미디어 생성 + 소셜 공유 플랫폼**
- 독자가 인상 깊은 문장/단락을 스크랩
- AI가 해당 텍스트를 분석하여 이미지, 비디오, 음성 생성
- 생성된 콘텐츠를 **"로그(Log)"**에 자동 저장 및 공유
- 다른 독자들의 시각화를 소설 본문 위에 **마커**로 표시
- 같은 장면에 대한 다양한 해석을 발견하고 감상

---

## 3. 타겟 사용자

### 주 타겟
- **웹소설 독자**: 판타지, 로맨스, 무협 등 장르 소설을 즐기는 독자
- 연령대: 10대 후반 ~ 30대
- 더 깊은 몰입 경험을 원하는 열성 독자

### 사용자 니즈
- 소설 속 인물, 배경, 액션 장면의 시각화
- 캐릭터 목소리, 배경음악 등 청각적 요소
- 개인화된 독서 경험
- 스크랩한 장면을 소장/공유하고 싶은 욕구
- 같은 소설을 읽는 다른 독자들과 교류
- 인기 장면, 다양한 해석 발견

## 4. 주요 기능

### 4.1 웹소설 플랫폼 (기본)
- 웹소설 읽기 뷰어
- 읽기 진행률 트래킹 (페이지/퍼센트)
- 북마크 및 독서 기록
- 작품 검색 및 추천
- 하단 네비게이션: 댓글 / 로그 / 설정

### 4.2 텍스트 스크랩 (Core)
- 독서 중 텍스트 드래그/선택하여 스크랩
- 스크랩한 텍스트 관리 (컬렉션)
- 작품별, 캐릭터별 스크랩 정리

### 4.3 AI 멀티미디어 생성 ⭐️ **킬러 기능**
#### 생성 프로세스
1. 텍스트 선택 후 플로팅 툴바 표시
2. 생성 타입 선택: 이미지 / 비디오 / 음성 / 메모
3. 바텀시트에서 옵션 확인 및 생성 시작
4. 로딩 후 결과 확인
5. "로그에 넣기" 또는 "다시 만들기"

#### 이미지 생성
- 장면, 캐릭터, 배경 이미지 생성
- 다양한 스타일 옵션 (사실적, 애니메이션, 일러스트 등)

#### 비디오 생성
- 짧은 장면 영상화 (5~15초)
- 액션 장면, 감정 표현 등

#### 음성 생성
- 캐릭터 대사를 음성으로 변환
- 내레이션 생성
- 배경 사운드/음악 추가

### 4.4 로그 (Log) ⭐️ **소셜 핵심 기능**

#### 로그란?
**해당 웹소설에 대한 모든 독자들의 시각화 콘텐츠가 모이는 공유 공간**

#### 로그 작동 방식
1. **생성 → 자동 저장**: 내가 문단을 시각화하면 해당 소설의 로그에 자동으로 push
2. **"로그 보기" 활성화**: 본문 위에 다른 독자들이 시각화한 위치를 **마커(원형 표시)**로 표시
3. **마커 클릭**: 해당 위치에서 다른 독자가 생성한 이미지/비디오/음성 감상
4. **반응**: 좋아요(하트), 저장, 댓글

#### 로그 갤러리
- 작품별 로그 모아보기
- 탭으로 필터링: 이미지 / 비디오 / 음성 / 메모
- "내 로그만 보기" 옵션
- 각 콘텐츠에 작성자 표시 (by 닉네임)
- 스크랩된 원본 텍스트 함께 표시

#### 소셜 기능
- 같은 장면에 대한 **다양한 해석** 비교
- 인기 장면 발견 (마커 밀도가 높은 구간)
- 독자 커뮤니티 형성
- 생성한 콘텐츠 공유 및 반응

## 5. 사용자 플로우

### A. 내가 시각화 만들기
```
1. 웹소설 읽기
   ↓
2. 인상 깊은 텍스트 드래그/선택
   ↓
3. 플로팅 툴바에서 AI 생성 타입 선택 (이미지/비디오/음성/메모)
   ↓
4. 바텀시트에서 옵션 확인 및 "만들기" 클릭
   ↓
5. 콘텐츠 생성 (로딩)
   ↓
6. 결과 확인 → "로그에 넣기" 또는 "다시 만들기"
   ↓
7. 로그에 자동 저장 (해당 소설의 로그에 push)
```

### B. 다른 독자 시각화 감상하기
```
1. 웹소설 읽기
   ↓
2. "로그 보기" 버튼 클릭
   ↓
3. 본문에 마커 표시됨 (다른 독자들이 시각화한 위치)
   ↓
4. 마커 클릭
   ↓
5. "{닉네임}의 로그" 바텀시트 열림
   ↓
6. 해당 독자가 생성한 이미지/비디오/음성 감상
   ↓
7. 좋아요, 저장, 댓글 등 반응
```

### C. 로그 갤러리 탐색
```
1. 하단 네비게이션 "로그" 탭 클릭
   ↓
2. 해당 소설의 모든 시각화 콘텐츠 표시
   ↓
3. 탭으로 필터링 (이미지/비디오/음성/메모)
   ↓
4. 원하는 콘텐츠 클릭하여 감상
   ↓
5. 좋아요, 저장, 공유
```

## 7. 핵심 차별점

1. **독서 + AI 생성의 완벽한 통합**: 읽다가 바로 생성, 끊김 없는 UX
2. **맥락 기반 생성**: 단순 프롬프트가 아닌 소설 전체 맥락 활용
3. **독자 중심**: 작가가 아닌 독자가 원하는 장면을 선택
4. **멀티모달**: 이미지, 비디오, 음성을 모두 지원
5. **텍스트 위의 시각적 레이어**: 본문 위에 마커로 다른 독자들의 시각화 표시
6. **집단 재해석 플랫폼**: 같은 소설, 같은 장면을 여러 독자가 다르게 해석
7. **발견(Discovery)**: 인기 장면, 창의적인 해석을 자연스럽게 발견
